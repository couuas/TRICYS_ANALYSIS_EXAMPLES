## 1. 全局敏感性分析（基于“性能指标总表”）

性能指标总表显示了四个关键性能指标随 `plasma.Fueling_Efficiency`（取值范围 0.01 到 0.9）的变化：

### 1.1 Startup Inventory（启动库存，单位：kg）
- **趋势**：随着 Fueling Efficiency 增加，启动库存显著减少。
  - 从 0.01 时的 202.47 kg 下降到 0.9 时的 2.69 kg。
  - 下降幅度极大（约 98.7%），表明 Fueling Efficiency 对启动库存有极强的影响。
- **敏感性**：**高度敏感**。效率每提高 0.1，库存平均减少约 22 kg（在低效区间）到 0.3 kg（在高效区间）。

### 1.2 Self Sufficiency Time（自持时间，单位：天）
- **趋势**：自持时间随 Fueling Efficiency 增加而缩短，但在高效区域（≥0.6）趋于稳定。
  - 从 0.01 时的无法自持（nan）到 0.1 时的 13.35 天，进一步缩短至 0.4 时的 7.87 天，并在 0.6–0.9 时稳定在 7.37 天。
- **敏感性**：**中度敏感**在低效区域（0.01–0.4），**不敏感**在高效率区域（≥0.6）。

### 1.3 Doubling Time（倍增时间，单位：天）
- **趋势**：随着 Fueling Efficiency 增加，倍增时间缩短。
  - 从 0.4 时的 486.87 天缩短到 0.9 时的 357.06 天。
  - 在 Fueling Efficiency < 0.4 时无法实现倍增（nan）。
- **敏感性**：**中度敏感**。效率每提高 0.1，倍增时间平均减少约 30 天。

### 1.4 Required TBR（所需氚增殖比）
- **趋势**：Required TBR 随 Fueling Efficiency 增加而降低，但在高效区域（≥0.6）趋于稳定。

## 2. 动态过程分析（基于“关键动态数据切片”）

### 2.1 初始阶段（前20个数据点）
- **所有配置**：初始库存均为 3500 g（3.5 kg）。

- **结论**：低 Fueling Efficiency 导致氚燃料迅速耗尽，无法维持反应；高效率则能减缓库存下降。

- **低效配置（C1–C4）**：库存持续为负，且数值极大（如 C1 达 -78953.6 g），表明严重短缺。
- **高效配置（C6–C10）**：库存在此阶段已转为正值（如 C10 为 812.085 g），表明系统开始净增长。
- **物理过程**：此阶段是氚库存由净消耗转为净增长的关键转折点。对于 Fueling Efficiency ≥ 0.4 的配置，系统在此之后开始实现自持；对于更低效率，库存持续短缺，无法自持。

### 2.3 结束阶段（后20个数据点）
- **低效配置（C1–C4）**：库存仍为负值（C1: -198895 g），严重短缺。
- **高效配置（C5–C10）**：库存显著增长（C10: 14917.7 g），表明系统成功实现氚增殖。
- **结论**：只有 Fueling Efficiency ≥ 0.4 的配置能在长期运行中实现氚燃料的净增长和自持。

## 3. 综合结论与建议

### 3.1 综合影响
- **Fueling Efficiency 的提高**：
  - **显著降低**启动库存需求（从 >200 kg 降至 <3 kg）。
  - **缩短**自持时间和倍增时间。
  - **降低**对氚增殖系统（TBR）的要求（从 >1.17 降至 ~1.015）。
- **临界点**：Fueling Efficiency ≈ 0.4 是一个关键阈值，低于此值系统无法实现氚自持。

### 3.2 利弊权衡
- **优点**：提高 Fueling Efficiency 可大幅降低对初始氚库存和氚增殖能力的依赖，使系统更易实现自持和倍增。
- **缺点**：过高的 Fueling Efficiency（如 >0.6）对性能提升的边际效益递减（Self Sufficiency Time 和 Required TBR 趋于稳定）。

### 3.3 设计与运行优化建议
1. **目标设定**：应将 Fueling Efficiency 至少提高至 0.4 以上，以确保系统能实现氚自持。
2. **资源优化**：在 Fueling Efficiency ≥ 0.6 后，进一步提高效率对降低 Startup Inventory 和 Doubling Time 仍有收益，但对 Self Sufficiency Time 和 Required TBR 的改善有限。因此，需权衡工程实现的难度与收益。
3. **系统设计**：
   - 若追求最小化启动库存 → 尽可能提高 Fueling Efficiency（接近 0.9）。
   - 若约束来自氚增殖能力（TBR）→ 将 Fueling Efficiency 提高至 0.6 即可满足 TBR ≈ 1.02 的要求。
4. **运行策略**：在启动阶段，若 Fueling Efficiency 较低，需准备大量初始氚库存，且系统可能无法实现自持。
